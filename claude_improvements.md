## Critical

### 1. Context stored in struct
**File:** `internal/crawler/crawler.go:34`

`CrawlRequest.Ctx context.Context` violates the project's own CLAUDE.md rule ("Never store a context in a struct"). A context stored in a channel-buffered request may be cancelled before the worker picks it up, or live longer than intended if the channel backs up.

**Recommendation:** Document this as a deliberate pragmatic exception (channel-dispatch requires it), or derive a fresh context with deadline in the worker.

---

### 2. `time.After` leak in hot loop
**Files:** `internal/watcher/scheduler.go:191`, `internal/mailer/mailer.go:263`

`time.After` creates a timer that is not GC'd until it fires. When the `select` takes the `wakeC` branch, the abandoned timer leaks. Over a long-running daemon with frequent feed registrations this is a slow but real memory leak.

**Recommendation:** Replace with `time.NewTimer` / `timer.Reset` pattern:
```go
timer := time.NewTimer(delay)
select {
case <-timer.C:
    s.dispatchDue(ctx)
case <-s.wakeC:
    if !timer.Stop() { <-timer.C }
case <-ctx.Done():
    timer.Stop()
    ...
}
```

### 3. Missing `rows.Err()` checks — [DONE]
**File:** `internal/db/sqlite/sqlite.go`

`GetFeeds`, `GetFeedsWithErrors`, and `GetUsersForFeed` iterate `*sql.Rows` but never check `rows.Err()` after the loop. If the database returns a mid-scan error, these functions silently return partial results. (`UnseenRecentItems` already does this correctly.)

**Fix:** Added `return ..., rows.Err()` after each `for rows.Next()` loop.

### 6. `Config.UseTLS` declared but never read
**File:** `internal/config/config.go:20`

### 7. `resyncFeeds` misses extraction strategy changes — [DONE]
**File:** `cmd/rss2go/daemon.go:109`

The change-detection comparison checks `URL`, `Title`, and `FullArticle` but not `ExtractionStrategy` or `ExtractionConfig`. If a user runs `feed update` to change extraction settings, the running daemon silently ignores the change until restart.

**Fix:** Expanded the comparison to include `ExtractionStrategy` and `ExtractionConfig`.

### 8. `Watcher.FormatItem` exported to serve one caller
**File:** `internal/watcher/watcher.go:320`

`runTestFeed` in `cmd/rss2go/feed.go` constructs a `Watcher` with nil store/crawler/mailer just to call `FormatItem`. This forces the export and creates a leaky abstraction — callers must pass nils for dependencies they don't need.

**Recommendation:** Extract `FormatItem` (and the content policies it depends on) into a standalone `Formatter` type or free function in the `watcher` package.

---

### 9. Metrics server goroutine leaks on shutdown
**File:** `internal/metrics/metrics.go:46`

The HTTP server for metrics is started in a goroutine and never stopped when the context is cancelled. `ListenAndServe` does not respect context cancellation — the goroutine and file descriptor leak on daemon shutdown.

**Recommendation:** Use `server.Shutdown(ctx)` in a select on `ctx.Done()`.

---

### 10. `AddUser` silently returns 0 on duplicate
**File:** `internal/db/sqlite/sqlite.go:283`

`ON CONFLICT DO NOTHING` + discarded `LastInsertId` error means callers receive `id=0` when the user already exists, with no indication that the row was not inserted.

**Recommendation:** On conflict, query and return the existing user's ID.
